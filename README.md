# 🎡 กงล้อมื้อเที่ยง (Lunch Wheel)

กงล้อสุ่มร้านอาหารมื้อเที่ยง เล่นบนมือถือ โทนตลาดกลางคืน มีเพลงประกอบและไฟวิ่ง

**เล่นเลย 👉 https://noboyband.github.io/foodwheel/**

## วิธีเล่น

กดปุ่มชมพูตรงกลางกงล้อ รอ 5 วินาที แล้วให้ดวงตัดสินว่าวันนี้กินร้านไหน

- **ตัดร้านนี้ออก** — ถ้าไม่ถูกใจ เอาออกจากกงล้อแล้วหมุนต่อ ตัดจนเหลือร้านเดียวมันจะประกาศให้เอง
- **แตะชิปชื่อร้าน** — ปิดร้านที่วันนี้ไม่เอา เช่นร้านปิด
- **ปุ่ม 🔊 มุมขวาบน** — ปิด/เปิดเสียงทั้งหมด
- ร้านที่ปิดไว้กับการตั้งค่าเสียงจะถูกจำไว้ให้ครั้งหน้า

เพลงจะเริ่มเองตอนกดหมุนครั้งแรก (เบราว์เซอร์ห้ามเล่นเสียงก่อนมีการแตะหน้าจอ)

## ร้านในกงล้อ

ร้านแม่ยาย · ร้านข้าวกรรมกร · ร้านต้มเลือดหมู นกพิราบ · ร้านตามสั่ง โต๊ะบอล ·
ร้านข้าวแกงน้ำพริกกะปิ · ชั้น 6 ttb · ร้านส้มตำ ซอย BTS · ร้านข้าวติดโรงแรม ·
ตึก Sun · ชั้น 2 ตลาดติด Top

แก้รายชื่อร้านได้ที่ตัวแปร `SHOPS` ใน [`src/template.html`](src/template.html)

## โครงสร้างไฟล์

| ไฟล์ | คืออะไร |
|---|---|
| `index.html` | หน้าเว็บที่ build แล้ว มีเพลงฝังอยู่ข้างใน — ไฟล์นี้คือตัวที่ GitHub Pages เสิร์ฟ |
| `src/template.html` | ซอร์สจริง ใช้ `__BGM_B64__` เป็นที่วางเพลง **แก้ที่ไฟล์นี้** |
| `src/loop.mp3` | เพลงประกอบที่แปลงเป็น MP3 แล้ว |
| `build.ps1` | สคริปต์รวม template + mp3 ออกมาเป็น `index.html` |

## วิธี build ใหม่หลังแก้โค้ด

แก้ `src/template.html` แล้วรัน

```powershell
.\build.ps1
```

สคริปต์จะอ่าน `src/loop.mp3` แปลงเป็น base64 แล้วยัดแทนที่ `__BGM_B64__`
ได้ออกมาเป็น `index.html` จากนั้น commit แล้ว push ตามปกติ

> **อย่าแก้ `index.html` ตรง ๆ** เพราะจะโดนสคริปต์ทับตอน build ครั้งถัดไป

## ทำไมต้องฝังเพลงเข้าไปในไฟล์

หน้านี้ออกแบบให้เป็นไฟล์เดียวจบ ไม่มี dependency ไม่ต้องมี build step ตอนรัน
เปิดจากที่ไหนก็ได้ เพลงจึงถูกฝังเป็น base64 ทำให้ `index.html` มีขนาดราว 685 KB
โหลดครั้งแรกบนเน็ตช้าอาจหน่วงนิดหน่อย แต่หลังจากนั้นเบราว์เซอร์จะ cache ไว้

เพลงเล่นผ่าน Web Audio API (ไม่ใช่ `<audio loop>`) เพื่อให้วนแบบไร้รอยต่อ
และหรี่/เร่งเสียงตอนหมุนได้
